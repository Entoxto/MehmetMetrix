# Инструкция по работе с данными

## Структура данных

- `products.json` — каталог изделий; генерируется из Excel и перезаписывается при парсинге
- `shipments.json` — партии с историческими ценами; генерируется из Excel и считается основным источником цен/статусов
- `money.json` — ручные финансовые строки, подписи и депозиты
- `meta.json` — метаданные обновления данных (время последнего парсинга)

## Что является источником правды

- Для поставок, статусов, размеров, материалов и цен источником правды является Excel / Google Sheet.
- `products.json`, `shipments.json` и `meta.json` не стоит редактировать вручную как постоянное решение: их перезапишет следующий парсинг.
- `money.json` можно редактировать вручную, это не автогенерируемый файл.
- Если нужно быстро проверить гипотезу локально, ручная правка `products.json` или `shipments.json` допустима только как временный обходной путь.

## Архитектура цен

**Цены хранятся в партиях**, а не в каталоге. Это позволяет:
- Сохранять историческую правду (партия 1 = $100, партия 2 = $110)
- Автоматически обновлять каталог — берётся цена из последней партии

## Как добавить новый товар в каталог

**Вариант 1 (рекомендуется):** добавьте строку в таблицу Excel (лист «Поставки») и запустите парсинг («Запустить с обновлением»). Товар появится в каталоге с id вида `auto-NNN`; `materials` теперь также подтягиваются из колонки `Состав` в Excel. При необходимости потом можно вручную доуточнить `photo`, `tags` и т.д.

**Вариант 2:** откройте `data/products.json` и добавьте объект в массив `products` (файл перезаписывается при каждом парсинге из Excel). Поле `price` не указывается — цена берётся из партий автоматически.

## Формат наименований

Все названия следуют формату: **`Изделие из материала — вариант`**

Примеры:
- `Жакет приталенный из кожи питона — чёрный глянцевый`
- `Брюки из замши — коричневый`
- `Дублёнка из мериноса — бежевый`

## Как временно вручную поправить партию

Правильный рабочий путь — менять Excel / Google Sheet и заново запускать парсинг.

Если нужна локальная проверка без немедленного обновления Excel:

1. Откройте файл `data/shipments.json`
2. Найдите нужную партию и добавьте объект в `rawItems`
3. Помните, что следующий парсинг перезапишет этот файл

```json
{
  "productId": "python-010",
  "overrideName": "Жакет приталенный из кожи питона — красный глянцевый",
  "price": 550,
  "sizes": { "xs": 3, "s": 2 },
  "status": "В производстве 🛠️"
}
```

**Обязательные поля:**
- `productId` — ссылка на товар в каталоге
- `price` — цена на момент партии (историческая)

**Опциональные поля:**
- `overrideName` — название для отображения (если отличается от каталога)
- `sample: true` — пометить как образец
- `quantityOverride: 1` — переопределить количество
- `note: "текст"` — примечание
- `paidPreviously: true` — оплачено ранее (не входит в сумму)
- `noPayment: true` — без оплаты (например, вернулось с ремонта)

> Статусы в JSON хранятся как текст из Excel. Старые кодовые варианты вроде `in_progress` поддерживаются только как обратная совместимость, но для новых данных их использовать не нужно.

## Как добавить фото

1. Поместите фото в папку `public/images/products/`
2. Назовите файл ТОЧНО как название товара с вариантом:
   - Пример: `Жакет приталенный из кожи питона — чёрный глянцевый.jpg`
3. Рекомендуемый формат: JPG
4. Рекомендуемый размер: 1200x1600px или больше

## Категории

Доступные категории:
- `Мех` — меринос, пони, кёрли, таскана
- `Замша` — замшевые изделия
- `Кожа` — кожа Hermes и др.
- `Экзотика` — питон и др. экзотические материалы

Каталог `data/products.json` при каждом парсинге **собирается заново** из Excel: в нём остаются только товары, встретившиеся в текущем файле. Категория выводится по названию (по корням): приоритет **Экзотика** (питон) > Кожа > Мех > Замша. Так «кожи питона» даёт Экзотику. Если ни один корень не найден, парсер считает это ошибкой и останавливается: категории `Прочее` в пайплайне быть не должно. **Размеры** (поле `sizes`) заполняются автоматически: объединение всех размеров, которые встречаются у этого товара в позициях поставок (из тех же данных, что уже есть в партиях); порядок — xs, s, m, l, xl, onesize. **Материалы** (поле `materials`) автоматически подтягиваются из колонки `Состав` в Excel: первая содержательная строка идёт в `outer`, строка с `Подкладка` — в `lining`, остальное сохраняется в `comments`.
