# Инструкция по работе с данными

## Структура данных

- `products.json` — каталог изделий; генерируется из Excel и перезаписывается при парсинге
- `shipments.json` — партии с историческими ценами; генерируется из Excel и считается основным источником цен/статусов
- `money.json` — ручные финансовые строки, подписи, депозиты и ручные доплаты в `Всего к оплате`
- `meta.json` — метаданные обновления данных (время последнего парсинга); используются в нижнем блоке интерфейса как дата последнего обновления

## Что является источником правды

- Для поставок, статусов, размеров, материалов и цен источником правды является Excel / Google Sheet.
- `products.json`, `shipments.json` и `meta.json` не стоит редактировать вручную как постоянное решение: их перезапишет следующий парсинг.
- `money.json` можно редактировать вручную, это не автогенерируемый файл.
- В `money.json`:
  - `deposits` — ручные строки для блока `Депозитов внесено`
  - `pendingManual` — ручные дополнительные строки для блока `Всего к оплате`
- `money.json` проверяется командой `npm run validate:data`: `amount` должен быть положительным числом, а строки `title` / `lines` — непустыми.
- Если нужно быстро проверить гипотезу локально, ручная правка `products.json` или `shipments.json` допустима только как временный обходной путь.

## Архитектура цен

**Цены хранятся в партиях**, а не в каталоге. Это позволяет:
- Сохранять историческую правду (партия 1 = $100, партия 2 = $110)
- Автоматически обновлять каталог — берётся цена из последней партии
- Себестоимость (`cost`) берётся из колонки N Excel только если курс списания в колонке J не равен `0`. При нулевом курсе колонка N может содержать только карго по формуле, поэтому такая строка не обновляет последнюю себестоимость каталога.

## Как добавить новый товар в каталог

**Вариант 1 (рекомендуется):** добавьте строку в таблицу Excel (лист «Поставки») и запустите парсинг («Запустить с обновлением»). Товар появится в каталоге с id вида `auto-NNN`; `materials` подтягиваются из колонки `Состав`, а `excelRows` хранит номера всех строк этой модели. Если подходящего JPG ещё нет, поле `photo` не создаётся и приложение показывает общую заглушку.

**Вариант 2:** откройте `data/products.json` и добавьте объект в массив `products` (файл перезаписывается при каждом парсинге из Excel). Поле `price` не указывается — цена берётся из партий автоматически.

## Формат наименований

Все названия следуют формату: **`Изделие из материала — вариант`**

Примеры:
- `Жакет приталенный из кожи питона — чёрный глянцевый`
- `Брюки из замши — коричневый`
- `Дублёнка из мериноса — бежевый`

Если в названии есть дополнительные скобки, размеры всё равно берутся из последней скобки:
`Куртка (капсула) из кожи — чёрная (XS-1, S-2)` → `xs: 1`, `s: 2`.

Если размеры пока не разбиты, но позиция уже должна участвовать в заказе (с известным общим количеством), в последней скобке можно поставить маркер `(на уточнении)`, а количество указать в колонке `G` «Кол-во в заказе». Пример: `Куртка из кожи — коричневая (на уточнении)` + G=`10`. Такая позиция получит флаг `sizesUnknown`, будет отображаться в Work с меткой «на уточнении» и не добавит фиктивный размер в каталог.

## Как временно вручную поправить партию

Правильный рабочий путь — менять Excel / Google Sheet и заново запускать парсинг.

Если нужна локальная проверка без немедленного обновления Excel:

1. Откройте файл `data/shipments.json`
2. Найдите нужную партию и добавьте объект в `rawItems`
3. Помните, что следующий парсинг перезапишет этот файл

```json
{
  "productId": "python-010",
  "overrideName": "Жакет приталенный из кожи питона — красный глянцевый",
  "price": 550,
  "sizes": { "xs": 3, "s": 2 },
  "status": "В производстве 🛠️"
}
```

**Обязательные поля:**
- `productId` — ссылка на товар в каталоге
- `price` — цена на момент партии (историческая)

**Опциональные поля:**
- `overrideName` — название для отображения (если отличается от каталога)
- `sample: true` — пометить как образец
- `quantityOverride: 1` — переопределить количество
- `note: "текст"` — примечание
- `paidPreviously: true` — оплачено ранее (не входит в сумму)
- `noPayment: true` — без оплаты (например, вернулось с ремонта)

`sample` — только метка образца. Если размеры или колонка G Excel задают количество больше 1, это количество сохраняется и участвует в сумме поставки.

> Статусы в JSON хранятся как текст из Excel. Старые кодовые варианты вроде `in_progress` поддерживаются только как обратная совместимость, но для новых данных их использовать не нужно.

> Ключи размеров в `rawItems.sizes` должны быть каноническими: `xs`, `s`, `m`, `l`, `xl`, `OneSize`. Неизвестный размер теперь считается ошибкой данных, а не превращается в `S`.

## Как добавить фото

1. Поместите фото в папку `public/images/products/jpg/`
2. Назовите файл ТОЧНО как название товара с вариантом:
   - Пример: `Жакет приталенный из кожи питона — чёрный глянцевый.jpg`
3. Рекомендуемый формат: JPG
4. Рекомендуемый размер: 1200x1600px или больше
5. Запустите «Запустить с обновлением» или `python Excel/parse_excel.py --auto`: парсер сам добавит `photo`, если файл найден.
6. Запустите `python scripts/convert_to_webp.py` для WebP-версии.

Фото не обязательно добавлять сразу. Без него модель остаётся валидной, а каталог показывает `__photo_pending`. При каждом запуске `npm run validate:images` выводится количество моделей без фото и список с номерами строк Excel.

## Категории

Доступные категории:
- `Мех` — меринос, пони, кёрли, таскана
- `Замша` — замшевые изделия
- `Кожа` — кожа Hermes и др.
- `Экзотика` — питон и др. экзотические материалы

Каталог `data/products.json` при каждом парсинге **собирается заново** из Excel: в нём остаются только товары, встретившиеся в текущем файле. Категория выводится по названию (по корням): приоритет **Экзотика** (питон) > Кожа > Мех > Замша. Так «кожи питона» даёт Экзотику. Если ни один корень не найден, парсер считает это ошибкой и останавливается: категории `Прочее` в пайплайне быть не должно. **Размеры** (поле `sizes`) заполняются автоматически: объединение всех размеров, которые встречаются у этого товара в позициях поставок (из тех же данных, что уже есть в партиях); порядок — xs, s, m, l, xl, onesize. **Материалы** (поле `materials`) автоматически подтягиваются из колонки `Состав` в Excel: первая содержательная строка идёт в `outer`, строка с `Подкладка` — в `lining`, остальное сохраняется в `comments`. **Строки Excel** сохраняются в `excelRows`, а **photo** добавляется только при наличии точного файла в `public/images/products/jpg/`.
